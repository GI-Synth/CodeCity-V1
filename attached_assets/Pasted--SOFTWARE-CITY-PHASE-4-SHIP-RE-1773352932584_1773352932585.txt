── 3A. VIEWPORT CULLING ────────────────────────────────────────────
In CityMap.tsx, compute the visible viewport rectangle from the current pan/zoom state. Only render buildings that intersect it.
const viewportBounds = useMemo(() => ({ minX: -pan.x / zoom, minY: -pan.y / zoom, maxX: (-pan.x + containerWidth) / zoom, maxY: (-
pan.y + containerHeight) / zoom, }), [pan, zoom, containerWidth, containerHeight])
const visibleBuildings = useMemo(() => allBuildings.filter(b => b.x + b.width > viewportBounds.minX && b.x < viewportBounds.maxX
&& b.y + b.height > viewportBounds.minY && b.y < viewportBounds.maxY ), [allBuildings, viewportBounds])
Render only visibleBuildings instead of all buildings. Always render roads (they’re cheap <line> elements). Always render all NPC
agents (max 8, always cheap). Always render district borders (just <rect> outlines).
── 3B. LEVEL OF DETAIL ─────────────────────────────────────────────
At zoom < 0.5: render buildings as simple colored rects only No text labels, no coverage bars, no status icons Districts show name
labels in large text only
At zoom 0.5–1.0: standard rendering Building labels visible, coverage bars visible Status icons visible ( ✓)
At zoom > 2.0: detailed rendering Show LOC number inside building rect Show complexity number in corner Coverage percentage as
text
Track zoom level in state. Switch rendering tier in useMemo.
── 3C. ROAD SIMPLIFICATION ─────────────────────────────────────────
At zoom < 0.5: hide individual file-to-file dependency roads Only show district-to-district connections (much fewer) At zoom 0.5–1.0:
show roads within same district + cross-district At zoom > 1.0: show all roads including within-building connections
This alone reduces SVG element count by ~60% at full city view.
