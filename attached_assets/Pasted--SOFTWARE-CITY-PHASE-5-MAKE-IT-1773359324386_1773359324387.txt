PATCH /api/agents/pause-all: summary: Pause or resume all non-retired agents requestBody: { action: “pause” | “resume” } response

“false_positive” } response 200: { agentId, verdict, truePositives, falsePositives, accuracy }
POST /api/knowledge/import: summary: Import knowledge entries from exported JSON requestBody: array of KnowledgeEntry OR {
entries: KnowledgeEntry[] } response 200: { imported: number, skipped: number, total: number }
GET /api/knowledge/search: summary: Server-side full-text search parameters: q (string), limit (integer, default 20) response 200: {
results: KnowledgeEntry[], total: number }
GET /api/repos: summary: List all loaded repositories response 200: array of RepoSummary schema
DELETE /api/repos/{slug}: summary: Delete a repository and all its data response 200: { deleted: boolean, slug: string }
POST /api/repos/{slug}/activate: summary: Set a repository as the active city response 200: RepoSummary
GET /api/shared/{token}: summary: Get a shared city snapshot response 200: SharedSnapshot schema response 404: { error: string }
POST /api/city/share: summary: Create a shareable snapshot of the current city response 200: { url: string, token: string }
GET /api/city/diff: summary: Get git diff between two commits parameters: path (string, optional), from (string), to (string) response
200: { rawDiff: string, stats: { additions, deletions, files } }
GET /api/city/snapshot: summary: Download current city as JSON response 200: CitySnapshot schema (application/json, attachment)

── 8A. PREVENT DUPLICATE AGENT TASKS ───────────────────────────────
Current bug: two agents can pick the same building target.
In agentEngine.ts: Add a Set<string>: activeTargets = new Set() In chooseTarget(): skip buildings whose ID is in activeTargets When
agent starts a task: activeTargets.add(buildingId) When agent finishes: activeTargets.delete(buildingId) Thread-safe because Node.js is
single-threaded
── 8B. ANALYSIS RESULT CACHING ─────────────────────────────────────
Analyzing the same file twice is wasteful. Add a simple in-memory cache to codeAnalyzer.ts:
const analysisCache = new Map<string, { result: FileMetrics, hash: string }>()
Before analyzing: compute hash of file content (simple djb2 or fnv1a) If cache has this hash: return cached result After analyzing: store
in cache
Cache size limit: 500 entries (LRU eviction — delete oldest when full) This is especially useful for the file watcher (most files
unchanged)
── 8C. WEBSOCKET MESSAGE BATCHING ──────────────────────────────────
When many agents are active, WS messages flood the client. Batch rapid messages into groups.
In wsServer.ts: Add a message queue: pendingMessages: WSMessage[] = [] Add a flush interval: setInterval(flushMessages, 50) ←
20fps max
Instead of broadcasting immediately: broadcastNPCMove(), broadcastBugFound(), etc. push to pendingMessages
flushMessages(): If pendingMessages is empty: return Group messages by type: npc_moves can be batched into one array Send: {
type: “batch”, payload: { npc_moves: […], bugs: […], … } } Clear pendingMessages
Update useWebSocket.ts to handle “batch” message type: Unpack and process each message in the batch
── 8D. MEMORY LEAK PREVENTION ──────────────────────────────────────
In agentEngine.ts: Agent intervals: store all interval IDs in a Map<agentId, NodeJS.Timeout> On agent retire/delete: clearInterval for
that agent’s ID On server shutdown (SIGTERM handler): clearInterval for ALL agent IDs
In wsServer.ts: On client disconnect: remove from clients Set immediately Periodically (every 30s): ping all clients, remove nonresponsive ones: Send { type: “ping” } Client should respond with { type: “pong” } Remove clients that don’t pong within 5s

 Software City
Transform any code repository into a living, explorable city. AI agents walk through your code, find bugs, and grow smarter.
[Screenshot or GIF here — add placeholder text: <!-- screenshot -->]
What is this?
Software City renders your codebase as a pixel-art city:
Each folder is a district
Each file is a building (height = lines of code)
AI agents walk the city testing your code
Bugs appear as fires. Fix them and the city heals.
Code health drives the city’s season — bad code = winter.
Quick Start (30 seconds)
Clone and run
git clone https://github.com/you/software-city
cd software-city
pnpm install
pnpm dev
# Open http://localhost:5173
No database setup. No API keys required. SQLite runs automatically.
Optional: Enable AI agents
# Install Ollama for local AI (free)
# https://ollama.com/download
ollama pull deepseek-coder-v2:16b
# Or use Groq (free tier, no local GPU needed)
echo "GROQ_API_KEY=your_key" >> .env
Docker (one command)
docker compose up
# Open http://localhost:8080
