function djb2(str: string): string let hash = 5381 for (let i = 0; i < str.length; i++) { hash = ((hash << 5) + hash) ^ str.charCodeAt(i) } return (hash >>>
0).toString(36)
In analyzeFile(filename, content): const hash = djb2(content) const cached = analysisCache.get(filename) if (cached && cached.hash === hash)
return cached.result
// … existing analysis …
// LRU eviction if (analysisCache.size >= MAX_CACHE_SIZE) { const oldest = […analysisCache.entries()] .sort((a, b) => a[1].timestamp -

── 7A. README.md (complete rewrite) ─────────────────────────────────
 Software City
Walk through your code like a city. AI agents find bugs. The city gets healthier as you fix them.
 <!-- Add screenshot here when ready -->
What is this?
Software City transforms any code repository into a living, explorable world:
Folders become districts
Files become buildings (height = lines of code)
AI agents walk the city and test your code automatically
Bugs appear as building fires
Code health drives the season — untested code = winter
Quick Start
Requirements: Node.js 20+, pnpm
git clone https://github.com/you/software-city
cd software-city
cp .env.template .env # optional — add API keys for AI features
pnpm install
pnpm dev
Open http://localhost:5173 and enter any GitHub URL. No database setup. No API keys required for basic use.
Optional: Enable AI Agents
Local AI (free, no internet after setup)
# Install Ollama: https://ollama.com/download
ollama pull deepseek-coder-v2:16b
Cloud AI (free tier available)
# Groq — free tier, very fast
echo "GROQ_API_KEY=gsk_..." >> .env
# or Anthropic Claude
echo "ANTHROPIC_API_KEY=sk-ant-..." >> .env
Docker (one command)
docker compose up
# Open http://localhost:8080
Features
City
Real AST analysis: TypeScript, JavaScript, Python, Go, Rust, Java
Live code metrics: LOC, cyclomatic complexity, import graph
Git history time-lapse: watch your codebase evolve
Shareable city URLs
AI Agents
5 agent roles: QA Inspector, API Fuzzer, Load Tester, Edge Explorer, UI Navigator
Local-first: Ollama handles 94%+ of tasks
Smart escalation: KB cache → Groq → Claude (only when stuck)
Learning knowledge base: every escalation answer saves forever
Agent reputation system: junior → mid → senior → principal
Dashboard
Live metrics: health score, coverage, bugs, escalation rate
Settings: configure AI provider, agent speed, analysis limits
Leaderboard: agent career stats and promotions
Architecture
artifacts/software-city/ React + Vite frontend
artifacts/api-server/ Express + TypeScript backend
lib/db/ Drizzle ORM + SQLite
lib/api-client-react/ TanStack Query hooks
lib/api-zod/ Zod schemas from OpenAPI spec
Contributing
See CONTRIBUTING.md for how to add new agent roles, analysis metrics, and city visualizations.

Contributing to Software City
Setup
git clone ...
pnpm install
pnpm dev
Project Structure
[brief table of directories and what they do]
Adding a New Agent Role
1. Add role to AgentRole enum in lib/db/src/schema/agents.ts
2. Add targeting bonus in agentEngine.ts chooseTarget()
3. Add card in Agents.tsx spawn dialog
4. Add thought templates in ollamaPrompts.ts
Adding a New Analysis Language
1. Add extension mapping in codeAnalyzer.ts detectLanguage()
2. Add complexity counting patterns in computeComplexity()
3. Add import extraction patterns in extractImports()
4. Test: load a repo with that language, verify metrics are non-zero
Adding a New City Building Type
1. Add FileType in lib/db/src/schema/types.ts
2. Add detection rule in codeAnalyzer.ts detectFileType()
3. Add color mapping in cityAnalyzer.ts (building.color)
4. Add visual in CityMap.tsx building renderer
Running Tests
pnpm run typecheck ← TypeScript validation (must be 0 errors) pnpm run build ← Production build test
PR Guidelines
pnpm run typecheck must pass before submitting
Describe what your change adds to the city metaphor
Prefer extending existing systems over adding new ones
