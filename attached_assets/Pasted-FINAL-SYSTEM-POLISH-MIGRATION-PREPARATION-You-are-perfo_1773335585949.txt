You are performing the final improvement pass on the Software City repository before it is exported out of Replit and moved to a local development environment using Visual Studio Code and GitHub Copilot.

Your task is to audit the entire repository, improve UX, performance, and developer observability, and prepare the project for long-term development. This must be done safely and without breaking any existing functionality.

---

## GLOBAL SAFETY RULES

Before implementing anything:

1. Scan the entire repository and understand the architecture.
2. Verify whether each requested feature already exists.
3. If a feature exists, improve it safely rather than replacing it.
4. If a feature does not exist, implement it using the current architecture.
5. Do NOT refactor unrelated systems.
6. Do NOT rename files unless absolutely necessary.
7. Do NOT change the existing architecture.
8. Maintain compatibility with the current Godot + FastAPI + WebSocket system.
9. Ensure performance targets remain valid:

   * 60 FPS rendering
   * < 2 second city reloads
   * scalable to 1000+ files
10. Prefer additive improvements rather than structural changes.
